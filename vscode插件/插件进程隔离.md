## 进程隔离

进程隔离是指操作系统为每个进程分配独立的虚拟地址空间，使不同进程之间无法直接访问彼此的内存空间、文件描述符、句柄等资源。

### 核心原理：

1. 每个进程有独立的内存空间（代码段、数据段、堆、栈）

2. 进程A不能直接读写进程B的内存

3. 防止一个进程崩溃影响其他进程

4. 提供安全性，防止恶意程序窃取数据

### 实现方式：

1. 硬件层面的页表保护

2. 操作系统的内存管理单元(MMU)

3. 内核态/用户态分离

### 进程隔离架构：

- Extension Host A/B: 独立的插件进程

- Main Thread: 主线程，作为通信枢纽

- 每个进程维护自己的 _commands Map

- 主线程用 CommandsRegistry 全局路由

- 命令执行两条路径：

    - 同进程: 命令在本进程 → 直接调用，无序列化开销
    - 跨进程: 命令在远程进程 → 两次 JSON 序列化 → 性能瓶颈
